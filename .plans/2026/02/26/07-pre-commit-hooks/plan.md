# Implementation Plan

> Source: `.plans/20260226/07-pre-commit-hooks/prompt.md`

## Overview

Add a pre-commit configuration to enforce file quality across the repository's Markdown, JSON, YAML, and shell files. This includes the pre-commit framework config, a markdownlint config to accommodate the existing file style, and updates to documentation.

## Architecture & Approach

The implementation uses the standard pre-commit framework with three hook sources, each targeting a specific file concern:

1. **pre-commit/pre-commit-hooks** — general-purpose checks (JSON, YAML, whitespace, EOF, merge conflicts, large files, private key detection)
2. **igorshubovych/markdownlint-cli** — Markdown linting with a custom config to suppress rules incompatible with the existing file style (long lines, inline HTML, YAML front matter)
3. **koalaman/shellcheck-precommit** — shell script analysis

This keeps the config minimal with no unnecessary hooks. All three hook sources are well-maintained and widely adopted. The pre-commit framework handles runtime dependencies (Node.js for markdownlint, etc.) automatically.

A `.markdownlint.yaml` config file is needed because the existing Markdown files use long lines, HTML comments (in plan/taskify templates), and YAML front matter (in SKILL.md files), all of which would trigger default markdownlint rules.

## Components

### Pre-Commit Configuration

**Purpose:** Central configuration that defines which hooks run on each commit.
**Inputs:** `.pre-commit-config.yaml` at the repository root, read by the `pre-commit` framework.
**Outputs:** Pass/fail status on each commit; automatic fixes for whitespace and EOF issues.
**Notes:** Uses three hook repos. The `pre-commit-hooks` repo covers JSON validation, YAML validation, trailing whitespace trimming, end-of-file fixing, merge conflict detection, large file prevention, and private key detection (`detect-private-key`). markdownlint handles Markdown linting. shellcheck handles shell script analysis.

### Markdownlint Configuration

**Purpose:** Suppress markdownlint rules that conflict with the existing file style.
**Inputs:** `.markdownlint.yaml` at the repository root, read by markdownlint-cli.
**Outputs:** Customized rule set for Markdown linting.
**Notes:** Must disable at minimum: MD013 (line-length) because existing files have long prose lines; MD033 (no-inline-html) because template files use HTML comments; MD041 (first-line-heading) because SKILL.md files start with YAML front matter. May also need MD024 (no-duplicate-heading) disabled since skill files reuse heading names like "Step 1", "Step 2" across different sections. The `.plans/` directory should be excluded from markdownlint via an ignore pattern, since plan files are working documents generated by skills.

## Project Overview

This phase delivered a set of targeted fixes and renames to the jinglemansweep plugin. It enhanced the PRD review skill to interview users about each requirement, renamed the validate skill for clarity, added model frontmatter to agent definitions, and standardised domain skill naming from `jms-skill-*` to `jms-role-*` (with a special case for the skill-authoring skill).

### Goals

- Improve the PRD review workflow by giving users explicit control over each requirement before finalising
- Make the validate skill name reflect that it validates task outputs (`jms-plan-task-validate`)
- Enable model selection in agent definitions via frontmatter
- Establish a consistent `jms-role-*` naming convention for domain/role skills, distinct from `jms-plan-*` pipeline skills

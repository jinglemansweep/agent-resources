## Project Overview

The jplan plugin had two naming and data-quality issues. First, the six `jp-persona-*` domain-role skills needed renaming to `role-*` to shorten invocations and align with their purpose as domain-role definitions. Second, timestamps written to `state.yaml` and task log files during pipeline execution were fabricated by the LLM rather than obtained from the system clock, producing incorrect dates, times, and durations.

### Goals

- Rename all `jp-persona-*` skills to `role-*` across directory names, file contents, and every reference throughout the codebase.
- Ensure all timestamps written by pipeline skills reflect actual wall-clock time obtained via shell commands, not LLM-generated values.
- Maintain full backward compatibility of the pipeline after both changes.

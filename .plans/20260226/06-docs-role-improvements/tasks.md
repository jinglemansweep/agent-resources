### Rules Section

- [x] **Add Rules section to `plugins/jinglemansweep/agents/jms-role-docs.md`** — Insert a new `## Rules` section between Domain Expertise and Quality Gates. This section codifies documentation accuracy and integrity principles. Include the following rules as a bullet list: (1) every claim in documentation must be verifiable in the codebase, (2) do not invent or assume features that do not exist, (3) do not remove documentation sections that are still accurate, (4) preserve the existing structure and style of each documentation file. These are documentation-quality principles, distinct from the Constraints section (which covers task-execution mechanics).

### Workflow Section

- [x] **Add Workflow section to `plugins/jinglemansweep/agents/jms-role-docs.md`** — Insert a new `## Workflow` section between Rules and Quality Gates. Use guidance-oriented steps (each step is a principle with brief explanation, not a rigid checklist). Include these steps as a numbered list: (1) **Audit the codebase** — thoroughly read source files, configuration, and tests to build a complete picture of the project's current state, features, and behaviour, (2) **Review existing documentation** — read all documentation files in the repository to understand current coverage, structure, and style, (3) **Remove stale content** — delete or update references to features, options, or functionality that no longer exist in the codebase, (4) **Add missing content** — document any features, configuration, components, or behaviour not yet covered, (5) **Verify accuracy** — cross-check documented examples, defaults, file paths, and option names against the actual implementation and fix discrepancies, (6) **Summarise changes** — provide a structured summary of documentation changes, categorised as: added (new sections or content), removed (stale content deleted), corrected (inaccuracies fixed or content updated). Do NOT include a "run quality checks" step — that is already covered by the Quality Gates section.

## Remaining Role Agent Workflows

### Python Agent

- [ ] **Add Workflow section to `plugins/jinglemansweep/agents/jms-role-python.md`** — Insert a new `## Workflow` section between Domain Expertise and Quality Gates (i.e. after the "Domain Expertise" section, before the "Quality Gates" section). Use guidance-oriented steps as a numbered list: (1) **Activate the virtualenv** — before any other work, find and activate the project's virtualenv (`.venv` or `venv`), or create one for new projects — this aligns with the agent's Mandatory Requirements, (2) **Understand the codebase** — read existing source files, tests, and configuration to understand the project's patterns, conventions, and architecture before making changes, (3) **Implement changes** — write code following the project's established conventions, structure, and style — match import patterns, directory layout, and naming, (4) **Write or update tests** — add or modify tests to cover the changes, following existing pytest conventions (fixtures, parametrize, conftest.py), (5) **Verify correctness** — run the project's test suite and quality gates to confirm all changes work as expected, (6) **Summarise** — report what was implemented, any decisions made during implementation, and any issues encountered.

## Open Questions

- [RESOLVED] Should `detect-private-key` be included as an additional safety hook? → **Decision:** Yes, include it. Low cost, good safety net for preventing accidental credential commits.
- [RESOLVED] Should the `.plans/` directory be excluded from markdownlint? → **Decision:** Yes, exclude it. Plan files are working documents generated by skills — linting them adds noise without value.

## Potential Blockers

- No blockers identified. All tools are well-established and compatible.

## Risks

- Existing files may have trailing whitespace or missing final newlines that the `end-of-file-fixer` and `trailing-whitespace` hooks will auto-fix on first run, producing a noisy initial commit. This is a one-time cost and not a real risk.

## Future Considerations

- If CI is added later, `pre-commit run --all-files` can be integrated into the pipeline.
- The markdownlint rule suppressions should be revisited if the file style evolves (e.g. if long lines are wrapped in future).
